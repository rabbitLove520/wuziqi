# 五子棋 AI 项目 (WPF + .NET 10)

这是一个基于 WPF 开发的 15x15 五子棋游戏，具备基础的启发式 AI 对手。

## AI 核心算法流程

AI (白棋) 的决策过程遵循以下步骤：

### 1. 局部搜索范围优化 (GetSearchRange)
为了提高计算效率，AI 不会遍历棋盘上所有的空位，而是：
- 扫描棋盘上所有已落子的位置。
- 仅将这些棋子周围 **2格范围内** 的空位加入搜索集合。
- 如果棋盘全空，则默认选择中心点 (7, 7)。

### 2. 启发式评分 (EvaluateMove)
对于搜索范围内的每一个候选空位，AI 会计算一个综合分值：
- **进攻分 (aiScore)**：假设 AI 在该点落子，能形成何种棋型。
- **防守分 (playerScore)**：假设玩家在该点落子，能形成何种棋型。
- **综合公式**：`Total = aiScore * 1.1 + playerScore` (AI 略微倾向于进攻，即在分值相同时优先尝试获胜)。

### 3. 棋型识别与评分 (ScorePattern)
AI 会在 4 个方向（横、竖、左斜、右斜）扫描连续的棋子数量及其两端的封堵情况 (Block)：
- **连五 (count >= 5)**：极高分 (10,000,000)，直接获胜。
- **活四 (count=4, block=0)**：极高分 (1,000,000)，两头通，必胜。
- **冲四 (count=4, block=1)**：高分 (100,000)，一头堵，需立即处理。
- **活三 (count=3, block=0)**：高分 (100,000)，若不封堵将变为活四。
- **眠三 (count=3, block=1)**：中等分 (1,000)。
- **活二 (count=2, block=0)**：低分 (1,000)。
- **以此类推...**

### 4. 最佳决策
- AI 比较所有候选位置的最终得分，选择分值最高的点落子。
- 如果有多个最高分位置，AI 会随机从中选择一个，增加下棋的多样性。
